리눅스가 어려운 이유부터 풀기

명령어를 외우기 전에 파일, 권한, 프로세스, 서비스, 네트워크가 서로 어떻게 연결되는지 먼저 봅니다.

리눅스 한 장 지도

파일

리눅스는 많은 것을 파일처럼 다룹니다. 설정도 파일, 로그도 파일, 장치도 /dev 아래의 파일처럼 보입니다.

1단계

파일시스템은 “주소가 있는 서랍장”입니다

Windows에서 C드라이브를 떠올리기 쉽지만, 리눅스는 /에서 시작하는 하나의 큰 나무로 보는 편이 쉽습니다.

/etc설정 서랍
/home사용자 방
/var계속 변하는 기록
/dev장치 입구
시험에서는 이렇게 나옵니다

/etc/passwd, /etc/shadow, /etc/fstab, /var/log처럼 “어느 파일이 무슨 역할인지”를 묻습니다.

2단계

권한은 “누가 무엇을 할 수 있나”입니다

권한은 소유자, 그룹, 기타 사용자에게 읽기·쓰기·실행을 나눠 주는 규칙입니다.

r읽기4
w쓰기2
x실행/진입1
755를 바로 이해하기

7은 4+2+1이라 소유자는 읽기·쓰기·실행 가능, 5는 4+1이라 그룹과 기타 사용자는 읽기·실행만 가능합니다.

3단계

프로세스는 “지금 실행 중인 일”입니다

프로그램 파일이 실행되면 프로세스가 됩니다. 각 프로세스에는 PID라는 번호가 붙습니다.

$ ps aux root 101 sshd user 825 bash $ kill -15 101
자주 나오는 함정

kill은 무조건 “삭제”가 아니라 프로세스에 시그널을 보내는 명령입니다. 기본 종료 요청은 TERM 15, 강제 종료는 KILL 9입니다.

4단계

서비스는 “계속 켜져 있어야 하는 일”입니다

SSH, 웹 서버, 데이터베이스처럼 백그라운드에서 계속 동작해야 하는 프로그램은 서비스로 관리합니다.

systemctl start ssh지금 켜기
systemctl enable ssh부팅 때 자동 켜기
journalctl -u ssh서비스 로그 보기
시험에서는 이렇게 구분합니다

“현재 실행”이면 start, “부팅 시 자동 실행”이면 enable, “상태 확인”이면 status입니다.

5단계

네트워크 문제는 순서대로 확인합니다

IP가 있는지, 길이 있는지, 포트가 열렸는지, 이름 해석이 되는지 순서대로 보면 덜 헷갈립니다.

ip addr내 주소 ip route가는 길 ping도달성 ss -tulpn열린 포트 digDNS
초보자가 자주 헷갈리는 점

ping 성공은 “웹사이트가 정상”이라는 뜻이 아니라, ICMP로 네트워크 도달이 된다는 뜻입니다.

공부 방식 참고

이 페이지는 초보자용 학습 사이트처럼 작은 주제, 상황 예시, 직접 눌러보는 확인 과정을 중심으로 만들었습니다. 명령어 의미는 GNU Coreutils, GNU Grep, systemd, Linux man-pages 같은 공식 문서 기준으로 정리했습니다.

개념을 먼저 잡고 명령어를 붙이면 암기가 훨씬 쉬워집니다.
