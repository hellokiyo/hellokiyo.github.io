오늘의 상식

Linux에서 everything is a file 이라고 말하는 이유

일반 파일뿐 아니라 장치, 파이프, 소켓도 파일처럼 다루는 인터페이스를 갖습니다. 그래서 /dev/sda 같은 장치 파일, /proc 같은 가상 파일시스템이 시험에 자주 나옵니다.

헷갈림 방지

/etc/passwd에는 실제 비밀번호가 없습니다

현대 Linux는 암호 해시를 보통 /etc/shadow에 둡니다. /etc/passwd는 계정, UID, GID, 홈, 셸 정보를 보는 파일로 기억하세요.

실사용

grep은 파일 찾기가 아니라 내용 검색입니다

로그에서 에러 줄을 찾거나 설정 파일 안의 특정 옵션을 찾을 때 씁니다. 파일 이름을 찾을 때는 find, 실행 파일 위치는 which가 더 맞습니다.

예시

tail -f /var/log/syslog

로그 파일의 마지막 줄을 계속 따라가며 봅니다. 서버에서 서비스가 죽거나 접속이 실패할 때 실시간 확인용으로 자주 씁니다.

출제 포인트

systemctl start와 enable은 다릅니다

start는 지금 바로 실행, enable은 부팅 시 자동 실행 등록입니다. 문제에서 “현재”인지 “부팅 시”인지 먼저 보세요.

함정

usermod -aG에서 -a를 빼면 위험합니다

기존 보조 그룹을 유지하며 새 그룹을 추가하려면 -aG를 같이 씁니다. -G만 쓰면 기존 보조 그룹 구성이 바뀔 수 있습니다.
